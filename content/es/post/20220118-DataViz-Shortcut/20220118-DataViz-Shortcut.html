Introducción

Muchas veces la razón por la que la gente se acerca al lenguaje R es porque han visto que es una gran herramienta para lograr mejores visualizaciones. Sin embargo, dado que R sigue siendo un lenguaje de programación, puede ser frustrante que para hacerlo tengas que lidiar con unas cuantas líneas de código para cargar, y preprocesar tus datos antes de empezar a crear la más simple visualización. Así que el objetivo principal de este post es proporcionar una manera más fácil de hacer frente a los pasos de la carga de sus datos a la visualización. Así que seguirás una guía para principiantes paso a paso para manejar tus datos hasta la visualización.

Lleva tus datos a R {datapasta}

R tiene diferentes opciones para cargar sus datos en R, ya sea utilizando la base de R, con la ayuda de Tidyverse u otros paquetes de datos, y con el uso de un IDE (Integrated Development Environment) como RStudio. Sin embargo, aunque con el tiempo cargar archivos en R se convierte en algo rutinario, para un usuario con poca experiencia puede ser una de las tareas que más frustración genera, y acaba por desanimar el aprendizaje. Por eso, utilizar un proceso intuitivo como copiar y pegar o arrastrar y soltar puede ser un muy buen amigo mientras se adquiere más experiencia en la escritura de código R.

Con esta idea nació el paquete de datos {datapasta}, que facilita el proceso de importación de datos en R, reduciendo el tiempo invertido y la frustración asociada.

¿Cómo funciona {datapasta}?

Lo primero que tenemos que hacer en RStudio es instalar {datapasta}: 

# Instalar el paquete de datos
install.packages("datapasta")

Lo segundo que debes hacer es reiniciar tú sesión de R (muy importante) y luego cargar {datapasta} con el siguiente código: 

# Cargar el paquete de datos
library(datapasta)

Luego debes asignar el operador y nombrar tu paquete de datos (data o el nombre que mejor te parezca) de la siguiente forma: 

# Nombrar tu set de datos
data <-

Ahora nos dirigiremos a donde tenemos los datos que nos interesa copiar y realizaremos la copia con el clic derecho y copy. Luego, cuando volvamos a nuestra sesión de R, colocaremos el cursor a la derecha del “<-”, y seguiremos los siguientes pasos:

Addins -> Browse Addins -> Paste as Tribble

Paste as Tribble Permite leer tablas delimitadas con tabulador, coma, pipe o punto y coma.

Para nuestro ejemplo de hoy utilizaremos la el set de datos penguins que puedes encontrar en el siguiente link:

penguins.csv

Grafica tus datos con {ggplot2}

{ggplot2} Es un paquete de visualización escrito para el lenguaje R, creado por Hadley Wickham en 2005 y basado en Leland Wilkinson’s “Grammar of Graphics”. Un esquema general para la visualización de datos, que separa un gráfico en componentes semánticos como escalas y capas.

Gráfico básico

Lo primero que haremos es revisar el conjunto de datos de pingüinos que acabamos de cargar en R, y seleccionar las variables que representaremos en nuestra primera figura.De todas las variables, para nuestro primer gráfico seleccionaremos “bill_length_mm” (longitud del pico en milímetros) y “bill_depth_mm” (profundidad del pico en milímetros) debido a que son variables continuas.

ggplot(data = data, 
       mapping = aes(x = bill_length_mm, 
                     y = bill_depth_mm)) + 
       geom_point()

Componentes de un gráfico en {ggplot2}

  • ggplot() : Es la función que crea un sistema de coordenadas -de forma general- a las que se incorporán capas. El primer argumento de la función es el set de datos (data=). Por si sola esta función no genera una capa.

  • data : El set de datos es una colección rectangular de datos con las variables (columnas) y sus observaciones/valores (filas) a mapear (penguins).

  • mapping : Es donde se especifican el conjunto de variables y observaciones que se “mapean” o asignan a las propiedades visuales a usar en el gráfico y que ejes asignar a estos valores (x= bill_length_mm,y= bill_depth_mm). Si no son especificadas -de forma general-, se deben indicar en cada capa adicionada a el gráfico.

  • geom_point() : Las capas del gráfico son incorporadas por las funciones o Geom. En este caso la función agrega una capa de puntos al gráfico. ggplot2 incluye más de 30 funciones geom, además de las desarrolladas por otros autores.

Elegir los Colores

El gráfico aterior solo nos permite ver la relación entre dos variables númericas continuas. Sin embargo, el set de datos tambien posee información asigada a grupos discretos como lo son la especie del pinguino y el sexo del individuo. Por esa razón ocuparemos otra propiedad visual como el color para incorporar una nueva capa de información en el gráfico. Para ello ocuparemos la opción “color” dentro de las variables mapeadas en “aes”.

ggplot(data = data, 
       mapping = aes(x = bill_length_mm, 
                     y = bill_depth_mm,
                     color = species)) + 
       geom_point()

Si no especificamos una escala de colores, R trazará cada nivel de la variable discreta mapeada según la paleta de colores cargada por defecto en ggplot2. Para evitar o modificar esto, utilizaremos la opción de incorporar una escala de colores manualmente con “scale_color_manual” y una lista de colores en su código HEX que debe tener el mismo número de valores que los niveles a mapear en la variable discreta. En este caso hay 3 especies presentes en el conjunto de datos de penguins, por lo que nuestra lista tiene 3 códigos HEX.

ggplot(data = data, 
       mapping = aes(x = bill_length_mm, 
                     y = bill_depth_mm,
                     color = species)) + 
       geom_point() +
   scale_color_manual(values = c("#393459","#F2AB27","#D96704"))

Elija el gráfico con los objeto geom

Una capa en ggplot2, combina datos, propiedades visuales, objetos geométricos (Geoms), funciones y/o transformaciones estadísticas (Stat), y de ajuste de la posición. En nuestro gráfico anterior hemos realizado un scatterplot. Por defecto en R hay que elegir un objeto “geom_…” para generar una visualización, pues de lo contrarío obtendremos una representación sin nuestros datos en ella.

En el gráfico anterior podemos ver que hay diferencias en la distribución de datos según la especie. Ahora eligiremos otra representación para mostrar de mejor manera la distribución de las variables continuas (bill_length_mm o bill_depth_mm) en base a los grupos discretos presentes (especies) en el set de datos. Por esa razón cambiaremos nuestro geom desde geom_point() a geom_boxplot() y generaremos dos plot, uno para cada variable continua (bill_length_mm o bill_depth_mm).

Bill length

ggplot(data = data, 
       mapping = aes(x = species, 
                     y = bill_length_mm,
                     color = species)) + 
       geom_boxplot() +
   scale_color_manual(values = c("#393459","#F2AB27","#D96704"))

Bill depth

ggplot(data = data, 
       mapping = aes(x = species, 
                     y = bill_depth_mm,
                     color = species)) + 
       geom_boxplot() +
   scale_color_manual(values = c("#393459","#F2AB27","#D96704"))
